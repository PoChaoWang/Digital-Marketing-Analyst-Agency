# Reference: Performance Drop Analysis

## 適用情境

使用者描述廣告成效異常或下滑，但未指定比較區間，例如：
- 「為什麼 Meta 成效變差了？」
- 「這週 ROAS 好像下降很多」
- 「某個 campaign 的 CPA 突然變高」

---

## 前置步驟：問題定位

在進入分析前，必須先完成問題定位。若以下任一項目不明確，在 Analysis Plan 中標記假設並請使用者確認後再繼續。

**1. 確認問題指標**

使用者描述的「成效變差」指的是哪個指標？

- 若使用者已明確指出（例如「CPA 變高」、「ROAS 下滑」），直接使用。
- 若描述模糊（例如「成效變差」、「表現不好」），預設檢查以下指標：Revenue、ROAS、CPA、CVR（GA4），以及 CTR、CPC（廣告平台）。在 Analysis Plan 說明此預設。

**2. 確認問題範圍**

- 是特定平台（Meta / Google）還是跨平台？
- 是特定 campaign 還是整體帳戶？
- 若使用者未指定，預設檢查所有可用平台與所有 active campaign，在 Analysis Plan 說明此預設。

**3. 推算比較區間**

根據問題時間點，自動推算前後期區間：

- **有明確時間點**（例如「這週」、「5/15 之後」）：以該時間點為後期起點，往回推對等天數作為前期。
- **無明確時間點**：預設以最近 7 天為後期，往前推 7 天為前期。

區間推算完成後，在 Analysis Plan 宣告前期 / 後期日期範圍，**請使用者確認後再繼續**。

---

## 分析執行

問題定位與區間確認完成後，完全依照 `reference/period-comparison.md` 的框架執行：

- 區間設定確認（長度一致、指定日期歸屬）
- Layer 1：Campaign 層級比較
- Layer 2：Creative 層級比較
- 資料來源規則（廣告平台 vs GA4）
- 輸出結構
- 注意事項

不重複定義上述規則，以 period-comparison.md 為單一來源。

---

## Performance Drop 專屬輸出補充

在標準 period-comparison 輸出結構之上，額外加入：

**問題定位摘要**（放在區間確認之前）：

```
## 問題定位

- **問題指標**：{使用者指定 / 預設檢查範圍}
- **問題範圍**：{平台、campaign 範圍}
- **比較區間推算依據**：{使用者描述的時間點 / 預設邏輯}
```

**下滑歸因標記**：

在 Inference 層，針對每個出現顯著變動的指標，標注最可能的歸因層級：

| 歸因層級 | 說明 |
|---------|------|
| `[Budget/Bid]` | Spend 或 CPM 變動為主要驅動 |
| `[Creative]` | 特定 creative 指標變動為主要驅動 |
| `[Audience]` | CTR 或 CVR 變動但 creative 無明顯變化 |
| `[External]` | 跨 campaign 同步下滑，無明顯內部原因 |

歸因標記為推論，必須標明信心程度與假設前提，不作為確定性結論。
