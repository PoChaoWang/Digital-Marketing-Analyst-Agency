# 互動指令政策

這份政策只定義指令的總則與路由方式。  
各指令的詳細流程要放在 `modes/commands/`。

## 原則

- 指令要少，而且要穩定。
- 預設用對話式引導，不要求使用者先理解 repo 結構。
- 每個指令都要有明確下一步，以及可讀的結果摘要。
- 詳細規則不要集中在這份檔案，避免 context 過重。

## 指令入口

- IDE 使用 `/ma-*` 指令。
- 終端機使用 shell 友善形式，例如 `ma start`。
- 兩種入口可以長得不同，但意義要一致。
- 如果沒有實作終端機入口，就不要假裝它已存在。

## 已定義指令

- `/ma-start`
- `/ma-method`
- `/ma-test`

## 路由規則

- `/ma-start` → 讀 `modes/commands/ma-start.md`
- `/ma-method` → 讀 `modes/commands/ma-method.md`
- `/ma-test` → 讀 `modes/commands/ma-test.md`

如果未來新增指令：

- 先在這份政策登記指令名稱
- 再新增對應的 `modes/commands/*.md`
- 如果有終端機入口，再定義相同語意的 shell 形式

## 共通輸出要求

- 回應要白話、簡短、可執行。
- 要說清楚有沒有改動。
- 要說清楚下一步。
- 不可輸出 secrets、tokens 或私人識別資訊。

## 維護原則

- 這份政策只管總則，不放完整流程細節。
- 各指令的步驟、輸出格式與例外處理，統一放到對應的 `modes/commands/` 檔案。
- 如果某個指令內容要改，只改對應 mode，除非指令名稱或路由本身也要變。
